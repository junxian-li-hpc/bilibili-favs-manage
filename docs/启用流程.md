# src/ 启用流程说明

本文档描述脚本从加载到就绪的完整启动链路。

## 入口：`src/main.js`

脚本由 `main()` 函数驱动，按以下顺序执行：

```
页面加载完成
  │
  ▼
① waitForPageLoad()          等待 document.readyState === 'complete'
  │
  ▼
② new FavoriteManager()      创建核心管理器实例
  │                           └─ 内部创建 BilibiliAPI（静态方法集合）
  │                           └─ 内部创建 CrossPageFlowManager（跨页面流程）
  │
  ▼
③ manager.initialize()       页面版本检测
  │                           └─ BilibiliAPI.detectPageVersion()
  │                           └─ 检查 DOM 中是否存在 SELECTORS.FAVORITES_SIDEBAR
  │                           └─ 不支持 → return，脚本退出
  │
  ▼
④ PageDetector.isOwnPage()   判断当前页面类型
  │                           └─ 对比 URL 中的 UID 与 Cookie 中的 DedeUserID
  │                           └─ 输出"我的收藏夹"或"他人的收藏夹"
  │
  ▼
⑤ crossPageFlow.resumeFlow() 检查是否有待恢复的跨页面流程
  │                           └─ 从 GM_getValue/localStorage 读取状态
  │                           └─ 有状态 → 执行创建/复制 → return（不显示面板）
  │                           └─ 无状态 → 继续
  │
  ▼
⑥ getAllFavoriteNames()       从 DOM 收集收藏夹列表
  │                           └─ BilibiliAPI.getCreatedFavorites()
  │                           └─ 只收集"我创建的收藏夹"分组，过滤掉"我追的"和"其他收藏"
  │
  ▼
⑦ new FloatingPanel()        创建浮动面板 UI
  │                           └─ setLoggerPanel(panel) 将日志输出绑定到面板
  │                           └─ panel.init() 创建 DOM 并挂载到 body
  │
  ▼
⑧ 挂载到全局                  window.BiliFavManager / window.BiliFavPanel
  │
  ▼
  就绪 ✅
```

## 关键分支：跨页面流程（他人页面 → 自己页面）

当用户在**他人收藏夹页面**发起批量复制时，脚本会自动走跨页面流程：

```
在他人页面点击"开始批量复制"
  │
  ▼
① 收集视频信息
  │  └─ collectVideoBvidsFromPage()  从 DOM 提取当前页视频 BV 号
  │  └─ batchGetVideoAids()          调用 B站 API 将 BV 号转为 aid
  │
  ▼
② 保存状态 + 跳转
  │  └─ StateManager.saveState()     保存任务列表 + 视频 aids
  │  └─ window.location.href         跳转到自己的收藏夹页面
  │
  ▼
③ 在自己页面恢复流程（页面加载时自动触发）
  │  └─ resumeFlow() → executeOnOwnPage()
  │     ├─ getOwnFavoritesList()     API 获取自己的收藏夹列表
  │     ├─ createFavoriteViaAPI()    创建缺失的收藏夹
  │     └─ addVideosToFavorites()    API 批量添加视频到目标收藏夹
  │
  ▼
  完成 ✅（无需返回他人页面）
```

**与旧流程的区别**：旧流程返回他人页面通过 DOM 对话框复制，但对话框只显示当前页面的收藏夹，
无法选择自己页面的。新流程全程通过 B站 API 操作，无需返回。

状态有效期 30 分钟，过期自动清除。

## 模块依赖关系

```
main.js
  ├─ FavoriteManager
  │    ├─ BilibiliAPI        (DOM 操作 + API 调用)
  │    │    ├─ DOM: 点击、选择、对话框交互（自己页面复制）
  │    │    └─ API: getCSRFToken, collectVideoBvids, addVideosToFavorites,
  │    │            getOwnFavoritesList, createFavoriteViaAPI（跨页面流程）
  │    ├─ CrossPageFlowManager
  │    │    ├─ StateManager   (GM_setValue/localStorage 持久化)
  │    │    ├─ PageDetector   (UID 对比判断页面归属)
  │    │    └─ BilibiliAPI    (API 调用：创建收藏夹、添加视频)
  │    └─ PageDetector
  ├─ FloatingPanel
  │    ├─ UIComponents       (DOM 元素工厂)
  │    ├─ EventHandlers      (拖拽、缩放、最小化)
  │    └─ PageDetector
  └─ logger                  (统一日志，支持面板输出)
```

## 入口条件

脚本正常运行需要满足：

- 页面 URL 匹配 `space.bilibili.com/{uid}/favlist`
- DOM 中存在新版 VUI 侧边栏元素（`SELECTORS.FAVORITES_SIDEBAR`）
- 用户已登录（Cookie 中有 `DedeUserID`）
