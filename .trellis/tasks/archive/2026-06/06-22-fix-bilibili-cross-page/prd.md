## Requirements

### P0 - 核心功能修复

#### 1. 跨页面新建收藏夹流程

**问题：** 在别人的收藏夹页面，"复制至"对话框中没有"新建收藏夹"选项（B 站平台限制）。

**解决方案：**

```
方案 A：自动跨页面流程（推荐）
1. 用户在浮动面板选择源和目标收藏夹（如果目标不存在）
2. 脚本检测当前页面是否为用户自己的页面
3. 如果在别人页面 && 需要新建收藏夹：
   a. 记录当前状态（URL、收藏夹名称）
   b. 跳转到用户自己的收藏夹页面
   c. 新建目标收藏夹
   d. 跳转回原页面
   e. 继续复制流程
4. 如果在自己页面 || 收藏夹已存在：直接复制

方案 B：提示用户手动创建（降级方案）
1. 检测是否在别人页面
2. 检测目标收藏夹是否存在
3. 如果需要新建 && 在别人页面：
   - 暂停流程
   - 显示明确提示："请先在自己的收藏夹页面创建目标收藏夹 [名称]"
   - 提供快捷链接跳转
   - 用户创建后回来继续
```

**实施方案：** 优先尝试方案 A；如果技术复杂度过高，降级为方案 B。

**Acceptance Criteria：**
- [ ] 脚本能检测当前页面所有者（自己 vs 别人）
- [ ] 在别人页面复制到新收藏夹时，能自动跳转或明确提示
- [ ] 跨页面流程完成后能正确返回并继续复制
- [ ] 用户体验流畅，状态不丢失

#### 2. 批量模式状态持久化

**问题：** 第一页复制完成后，批量模式退出，导致第二页无法全选。

**解决方案：**

- 如果批量模式已退出，重新点击"批量操作"按钮
- 添加状态验证：检查全选按钮是否可见

**Acceptance Criteria：**

## Technical Constraints

1. **纯客户端脚本**：不能修改 B 站服务端，所有逻辑在浏览器运行
2. **保持兼容性**：适配新版 B 站页面（VUI 组件库）
3. **模块化架构**：ES6 模块 + Rollup 打包
4. **油猴脚本规范**：最终产物必须符合 UserScript 格式
5. **避免反爬虫**：操作间需要合理延迟

## Unknowns & Risks

### 技术风险

1. **跨页面状态保存**
   - 需要使用 `localStorage` 或 `sessionStorage`
   - 状态恢复可能失败
   - 缓解：添加重试机制和用户确认

2. **页面跳转检测**
   - 如何判断当前页面是否为用户自己的页面？
   - 可能的方案：比对 URL 中的 UID 与登录用户的 UID
   - 需要探测如何获取登录用户 UID

3. **模块化 UI 集成**
   - 原始 UI 代码紧密耦合
   - 拆分可能引入新 bug
   - 缓解：先迁移，再测试完整流程

4. **构建产物体积**
   - 模块化后代码量增加
   - 需要确认是否需要压缩（terser）
   - 目前 rollup.config.js 中 terser 已注释

### 用户体验风险

1. **跨页面跳转用户感知**
   - 自动跳转可能让用户困惑
   - 缓解：显示明确的提示和进度
   - 可能需要用户确认再跳转

2. **操作耗时较长**
   - 大收藏夹（1000+ 视频）需要数分钟
   - 需要清晰的进度反馈
