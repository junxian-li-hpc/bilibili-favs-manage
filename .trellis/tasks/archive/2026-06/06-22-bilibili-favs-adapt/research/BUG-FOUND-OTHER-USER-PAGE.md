# Bug: 别人收藏夹页面无法新建收藏夹

## 问题描述

在别人的收藏夹页面执行批量复制时，点击"复制至"按钮后弹出的对话框中**没有"新建收藏夹"选项**。

## 测试日志

```
2026-06-22 01:31:33:开始复制收藏夹[卦]到[卦],一共1页
2026-06-22 01:31:33:正在保存第1页--------------
2026-06-22 01:31:33:点击[全选]按钮
2026-06-22 01:31:34:点击[复制至]按钮
2026-06-22 01:31:34:创建新的收藏夹：[卦]
2026-06-22 01:31:34:点击[新建收藏夹]按钮
2026-06-22 01:31:34:Error: 找不到新建收藏夹按钮
```

## 根本原因

这是 B 站的权限限制：
- 在**自己的收藏夹页面**：弹窗有"新建收藏夹"选项
- 在**别人的收藏夹页面**：弹窗只显示自己已有的收藏夹列表，无法新建

## 建议的流程优化

### 当前流程（有问题）
```
1. 在别人的收藏夹页面
2. 选择源收藏夹 → 批量操作 → 全选 → 复制至
3. 尝试新建目标收藏夹（失败！）
```

### 优化后的流程
```
1. 用户在浮动面板选择要复制的收藏夹
2. 记录所有源收藏夹名称和目标收藏夹名称
3. 跳转到用户自己的收藏夹页面
4. 检测并新建所有缺失的目标收藏夹
5. 返回别人的收藏夹页面
6. 执行批量复制（此时目标收藏夹都已存在）
```

## 其他发现的问题

1. **第二页全选失败**：第一页复制完成后，批量模式可能自动退出
2. **批量操作按钮消失**：执行一次复制后，页面状态改变，批量操作按钮可能不再显示

## 优先级

- **P0 (Critical)**：实现跨页面流程，先在自己页面创建收藏夹
- **P1 (High)**：确保每页操作前重新进入批量模式
- **P2 (Medium)**：添加页面状态检测和恢复机制
